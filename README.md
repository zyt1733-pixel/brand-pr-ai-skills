# 舆情危机对外声明 · crisis-official-statement

一个面向品牌公关从业者的 AI 技能。当企业遭遇舆情危机、已经决定要对外发声时，本技能帮助完成四件事：为声明定调、给出发布建议、写出可以直接使用的声明全文、发布前逐维度检验。

## 它和"AI 帮我写一份声明"的区别

不是把声明写漂亮，而是把该想清楚的先想清楚：

- **先定调再动笔**：这份声明要把事件讲成一个什么故事、企业在其中是什么角色。不定调，就是跳进对方的框架自证清白，越辩越黑。
- **认错力度依据事实**：确有过错就充分认错，只有部分责任就只认该认的部分，没有责任时表达关切但不道歉。不盲目揽责，也不冷硬甩锅。
- **姿态谦卑，先情后理**：先接住公众的情绪，再讲事实和道理，法律表述放在最后。
- **道歉伴随代价**：只有态度没有代价的声明，公众不会买账。整改、退赔、处理责任人，代价具体到可验证。
- **发布前用六个维度检验**：态度、速度、高度、气度、尺度、温度，逐项过，避免"发出去反而更糟"。

## 触发场景

用户说以下这些话，都可以调用本技能：

- 帮我写一份官方声明 / 致歉信 / 澄清声明
- 这件事该怎么回应
- 谁出面、什么时候发、发在哪
- 帮我写一封危机中的创始人内部信
- 这份声明发布前帮我把把关

## 交付内容

调用后依次给出五部分：

1. **定调建议**：这份声明讲一个什么故事、企业是什么角色。
2. **发布建议**：谁来发、发在哪、什么时候发，每条附判断依据，注明"请结合实际情况灵活运用"。
3. **声明全文**：可直接使用，商业决策处以【】留空由企业填写。
4. **六个度检验表**：逐项标注通过或需修改，说明正文已根据检验调整。
5. **后续提醒**：声明只是危机沟通的开始，后续需要持续通报进展；发布前建议再请一位未参与处理的同事通读把关。

## 边界

本技能提供专业判断和声明初稿，最终决策与发布责任在使用者。不代替企业决定赔偿金额、整改天数、处分方式和高管署名；不代替法律意见，涉及诉讼、监管立案等事项请由法务把关。所有参考范文均已做匿名化处理，不指向任何具体企业、人物、时间和地点。
