# 舆情危机专业案例成稿规范

## 目录

1. 成稿目标与角色
2. 写作、证据与去重总则
3. 八模块标准化框架
4. 分析深度要求
5. 语言风格规范
6. 成稿质量检查

## 1. 成稿目标与角色

以资深危机传播研究者、企业公关实务专家和商业案例作者的视角写作。最终产出不是新闻摘要，也不是证据表格的机械拼接，而是一篇建立在完整事实底稿之上的专业案例分析。

成稿应达到以下标准：

- 逻辑严谨，论述充分，信息密度高；
- 语言丰满、形象、具有可读性，但不口语化；
- 兼具事实还原、传播分析、商业洞察和危机管理复盘；
- 经人工复核与编辑后，可以作为舆情危机处置书籍中的独立案例章节；
- 所有事实、数据、引语和外部观点均可追溯，所有原创分析均能回到事实和证据。

## 2. 写作、证据与去重总则

### 2.1 零杜撰

- 不得编造人物、时间、数据、声明、内部会议、心理活动、媒体报道、监管动作、财务影响或来源链接。
- 找不到可靠信息时，明确写“截至检索截止时间，未检索到可靠公开资料”。
- 不为填满章节而补写情节，不把合理猜测伪装成事实。

### 2.2 事实、观点与分析分层

- 事实使用事实还原协议中的 `[F]`、`[C]`、`[M]`、`[U]` 等证据状态。
- 媒体评论、专家意见和公众表达均归属于具体主体，不写成作者自己的事实判断。
- 原创分析标 `[I]` 或使用“从现有公开信息看”“一种可能解释是”“分析认为”等边界表述，并引用支持它的事实编号或来源编号。
- 未经官方认定的违法、欺诈、造假、侵权等指控，必须使用“被指”“某方称”“媒体报道”“通报称”等准确措辞。

### 2.3 链接与引用

- 每个关键事实、重要数据、外部观点和高风险表述都必须有来源。
- 时间线、回应表和观点表的“来源”列应直接提供可点击原文链接，不只写媒体名称。
- 叙述正文可以使用 `[S01]` 形式的来源编号，段落末尾紧邻标注；文末来源索引给出标题、发布主体、发布时间、原始链接和访问日期。
- 企业每一次重要回应尽量使用企业官网、官方账号或法定披露原文；原文不可访问时，注明限制，并使用可信媒体留存或公开存档交叉验证。
- 引用原话只使用核实所需的短摘，保持上下文和原意，不大段复制报道、声明或评论文章。

### 2.4 表格与正文分工

- 表格负责时间、主体、数据、回应次数、观点差异和来源等精确信息。
- 正文负责解释事实之间的关系、传播阶段变化、企业行动逻辑和管理启示。
- 每张重要表格之后至少有一段综合分析，不能以表格代替论述。
- 主文不能充斥证据标签和技术字段；对争议事项保留标签，对稳定背景信息使用自然叙述和来源编号。

### 2.5 单一成稿与去重

- 事实还原、分析复盘和专业成稿是同一份报告的内部工序。除非用户明确要求中间稿，默认只输出一份统一成稿，不显示“阶段 A／B／C”标题。
- 案例背景只在“案例基本信息”中完整介绍一次。
- 事件经过只在“详细时间发展脉络”中完整叙述一次。
- 企业或当事方回应的事实与短摘，在时间线或回应表中选择一处完整呈现；另一处只保留必要摘要和交叉引用。
- “危机应对措施评估”只增加回应时效、内容、渠道、证据、履约与效果评价，不重写时间线。
- “深层原因分析”只解释可验证的原因与备选解释，不再罗列事件节点。
- “影响与启示”只写已发生的变化、因果边界和可迁移结论，不重复回应评价。
- 同一数据、引语和案例细节原则上只完整出现一次；后文需要使用时，以日期、事实编号或“如前述回应所示”等方式回指。
- 同一来源链接在正文首次使用时提供，并在文末来源索引收录；不在多个章节反复粘贴。
- 两张表格如果超过一半字段或信息重合，必须合并，或将其中一张改为简表。
- 每个后续章节必须增加新事实、新判断或新启示。如果一段只是对前文换一种说法，删除或并入首次出现的位置。

## 3. 八模块标准化框架

### 一、案例基本信息

#### 1. 案例名称

使用“年份＋公司／品牌＋事件核心词”，例如：`2026年××品牌产品质量与信任危机`。

#### 2. 事件概述

| 项目 | 内容 | 信息状态 | 来源 |
|---|---|---|---|
| 爆发时间 | YYYY年MM月DD日；能确认时补充具体时刻与时区 |  |  |
| 最早公开记录 | 平台、账号或媒体、标题、发布时间 |  |  |
| 核心争议 | 用2—3句话概括事实争议与价值冲突，不提前定责 |  |  |
| 主要主体 | 企业、品牌、子公司、当事人、平台、监管部门等 |  |  |
| 影响范围 | 地域、受影响人群、业务范围、传播平台 |  |  |
| 舆情热度 | 热搜、阅读量、搜索指数等；无可靠数据则说明 |  |  |
| 当前进展 | 调查、处罚、判决、整改、持续争议或结束状态 |  |  |

#### 3. 企业／品牌背景

至少研究：

- 成立时间和关键发展阶段；
- 核心产品或服务；
- 商业模式、收入来源、渠道关系和主要成本逻辑；
- 市场地位、用户或门店规模、竞争优势；
- 融资、上市和公开财务情况；
- 与本事件直接相关的制度、供应链或业务背景。

背景信息必须服务于理解危机。不要堆砌与事件无关的创始人故事、融资新闻或企业荣誉。

#### 4. 开篇综合叙述

在表格之后，用完整段落回答：发生了什么、为什么形成公共争议、事件触及哪些利益相关方、当前能够确认到什么程度。语言应具有叙事吸引力，但不能虚构场景。

### 二、详细时间发展脉络

采用时间轴表格：

| 事件发生时间 | 信息发布时间 | 事件节点 | 具体描述 | 阶段性影响／传播意义 | 信息状态 | 原文链接 |
|---|---|---|---|---|---|---|

按公开资料实际存在情况纳入：

- 首发媒体、账号和最早可核实内容；
- 专业媒体原创报道和关键调查；
- 企业、品牌、高管、客服或门店的每一次重要回应；
- 党媒、央媒和重要行业媒体介入；
- 监管部门发声、立案、调查、处罚或司法进展；
- 微博、抖音、小红书、微信、百度等平台热搜或话题节点；
- 股价、市值、投资人、合作方、渠道和业务变化；
- 产品下架、暂停服务、人员处理、制度整改和第三方验证；
- 后续更正、反转、新证据和长期修复动作。

要求：

- 每个节点写明具体日期，只有公开资料支持时才写具体时刻。
- 分开记录事件发生时间和信息发布时间。
- “影响评估”必须说明依据；属于分析判断时标 `[I]`，不能把时间先后写成因果。
- 某类重要节点确实不存在公开记录时，写“暂无可靠公开记录”，不要虚构或完全隐去研究缺口。

时间表之后，将事件划分为触发期、扩散期、回应期、处置期、修复期等实际存在的阶段，分析每一阶段的议题变化和推动因素。

### 三、多维度观点梳理

#### 1. 权威与重要媒体观点

通常选择3—5家对事件事实、价值判断或后续走向有实质贡献的媒体；有效来源不足时，不为凑数量降低标准。

| 媒体 | 媒体类型／定位 | 报道或评论标题 | 核心立场 | 评价角度 | 发布时间 | 原文链接 |
|---|---|---|---|---|---|---|

分析媒体是在核查事实、批评企业、讨论行业、表达公共价值还是推动监管。区分原创报道、社论评论、转载和商业合作内容。

#### 2. 社交媒体舆情

至少说明：

- 主要传播平台和话题标签；
- 愤怒、失望、恐惧、质疑、支持、中立等主要情绪及其背后诉求；
- 代表性观点的差异，而不是只摘最激烈评论；
- 可核实的阅读量、互动量、搜索指数或舆情报告数据；
- 数据平台、时间范围、关键词、样本量、采样和分类方法；
- 平台用户与活跃发帖者不能代表全部公众的限制。

代表性网友评论只短摘3—5条，标注平台和大致时间，默认隐藏普通用户ID和头像信息。

#### 3. 专家／KOL分析

| 专家／KOL | 机构、职称与专业领域 | 核心观点 | 分析角度 | 发布平台与时间 | 利益关系或局限 | 原文链接 |
|---|---|---|---|---|---|---|

专家身份必须核查。观点应准确转述，不因专家知名度而自动判定为事实。

#### 4. 观点综合

在三类观点之后，分析不同群体对“事实是什么、责任在哪里、企业应该做什么”的分歧。指出哪些观点得到新证据支持，哪些仍停留在立场表达。

### 四、危机应对措施评估

#### 1. 短期回应管理

企业每一次重要回应单独列出：

| 回应次数 | 发布时间 | 距最早公开记录时长 | 发声主体 | 渠道 | 核心内容与短摘 | 原文链接 | 证据化评估 |
|---|---|---|---|---|---|---|---|

评估维度：

- 回应速度是否与危机阶段和信息核实难度匹配；
- 是否回应核心事实和公众关切；
- 是否体现受影响者视角和具体补救；
- 是否说明调查方法、责任主体、行动时限和后续更新；
- 不同回应之间是否存在口径冲突或事实变化；
- 承诺是否有后续执行证据。
- 若声明承诺定期更新、限期完成、召回或补偿，单独核对其频率、期限、覆盖范围与实际履约记录，不得只评价承诺文本。

不要使用“真诚”“傲慢”等无法验证的动机词，改写为可观察的内容分析。

#### 2. 沟通策略

分析：

- 高管、创始人、总部、区域公司、门店或客服分别承担了什么沟通角色；
- 是否接受媒体专访、召开新闻发布会或通气会；
- 回应渠道是否覆盖主要利益相关方；
- 是否主动提供证据、第三方材料和更新机制；
- 是否存在删帖、沉默、转移议题、切割责任或过度承诺等可观察风险。

#### 3. 立即整改行动

核查暂停服务、下架产品、召回、退款、人员处理、用户补偿、检测调查、供应链调整、绩效或管理机制改革。严格区分：宣布计划、开始执行、企业声称完成、外部验证完成。

#### 4. 中长期品牌修复

##### 价值观重构

- 企业是否重新解释使命和价值观；
- 是否用内部制度、文化建设、培训、考核和问责支撑；
- 价值观口号是否转化为可验证行动。

##### 信任重建

- 信息公示、进度报告、开放日、用户沟通与申诉机制；
- 第三方检测、认证、审计、专家或权威机构合作；
- 重要媒体正面报道的内容性质、独立性和实际信息增量。

##### 合规与监管

- 如何响应监管要求并完成整改；
- 是否取得相关认证；
- 是否参与行业标准、团体标准或治理机制建设。

长期修复必须核查实际执行与结果，不能把企业宣布的计划直接写成修复完成。

### 五、深层原因分析

先写直接诱因，再分别研究以下三层：

#### 1. 商业模式

研究核心盈利逻辑、规模扩张、成本控制、渠道依赖、供应链结构、平台机制与用户或社会利益之间是否存在张力。

#### 2. 管理机制

研究治理结构、授权链条、绩效激励、质量控制、客服升级、内部监督、合规审查和危机预案是否存在有证据支持的漏洞。

#### 3. 行业环境

研究监管规则、行业标准、竞争压力、供应链、平台生态和行业共性问题。区分行业共性与企业个体责任。

使用下表约束分析：

| 原因层 | 分析判断 | 支持事实与来源 | 反向证据／替代解释 | 置信度 | 仍需验证 |
|---|---|---|---|---|---|

原因分析可以深入，但不能“看到一次事故就认定文化有毒”，也不能“看到股价下跌就认定由危机导致”。

### 六、影响与启示

#### 1. 对企业自身的影响

根据证据研究：

- 品牌声誉、用户信任和社会评价；
- 用户、销量、营收、利润、门店、下载量、渠道和合作关系；
- 股价、市值和投资人反应；
- 人才、组织、管理制度和合规成本；
- 短期舆情波动和长期经营影响。

#### 2. 对行业发展的影响

根据证据研究：

- 监管政策、执法重点和制度变化；
- 行业标准、产品标准和服务规范；
- 行业竞争格局、上下游与产业链；
- 消费者和社会对行业的信任；
- 个案是否暴露行业共性问题，以及其他企业是否调整做法。

使用表格呈现：

| 影响层面 | 可核实变化 | 时间与比较基准 | 因果边界 | 分析结论 | 来源 |
|---|---|---|---|---|---|

没有公开数据时，明确写“暂无可靠公开数据”，不使用“损失惨重”“行业震动”等空泛结论。

表格之后形成完整论述，回答哪些动作有效、为什么有效、哪些做法失当、问题发生在哪个环节、类似企业在什么条件下可以借鉴。避免把经验写成“及时回应、真诚沟通”一类正确但无用的空话。

### 八、重要链接汇总

在文末按类别汇总所有重要链接：

#### 官方声明与回应

`序号｜标题｜发布主体／平台｜日期｜原文链接｜访问状态`

#### 党媒、央媒与重要媒体

`序号｜标题｜媒体｜日期｜原文链接`

#### 商业媒体深度报道

`序号｜标题｜媒体／作者｜日期｜原文链接`

#### 监管、司法与政府通报

`序号｜文件或通报名称｜发布部门｜日期｜原文链接`

#### 社交媒体话题与舆情数据

`序号｜话题／报告｜平台或机构｜时间范围｜数据口径｜链接`

#### 专家／KOL观点

`序号｜观点标题或内容｜专家与机构｜平台｜日期｜链接`

#### 企业与行业背景

`序号｜资料标题｜来源｜统计年份｜链接`

#### 其他重要参考

记录不能归入上述类别但确有证据价值的资料。

## 4. 分析深度要求

### 表象到本质

从具体争议继续追问：事件为何在此时、以这种形式爆发？企业业务和管理中有什么可验证的结构性条件？不得用抽象的“价值观问题”代替证据。

### 个案到普遍

判断该问题是否为行业共性。至少寻找同类制度、监管材料、行业报告或其他企业公开案例；没有比较材料时，不上升为行业结论。

### 短期到长期

区分即时热搜和持续声誉，区分声明发布和整改兑现，区分短期业务波动和长期经营结果。

### 企业到社会

分析事件对消费者信任、公共价值、监管治理、行业生态和产业链的影响，但只写公开证据能够支持的范围。

## 5. 语言风格规范

### 基本要求

- 客观中立：避免情绪化定性，基于事实分析。
- 逻辑严谨：保持“论点—事实依据—分析过程—结论边界”的完整链条。
- 层次清晰：使用分级标题和阶段性小结，不用大段材料堆积。
- 语言丰满：解释事实之间的联系和意义，不只罗列节点。
- 专业可读：不口语化，不写成公文，也不堆砌行业术语。
- 重点突出：关键数据、正式认定、回应转折和核心结论可适度加粗。

### 形象化与适度幽默

- 可以使用准确的类比、隐喻和轻度幽默解释传播机制、管理矛盾或行业现象。
- 类比必须帮助理解，不能替代证据。
- 不拿受害者、伤亡、疾病、未成年人、普通员工或尚未查明的争议开玩笑。
- 不讽刺当事人的人格、智力或动机，不使用侮辱性绰号。
- 在事实表格、官方通报、法律争议和伤害描述中保持完全克制。

### 禁止的写法

- 不写“企业显然故意”“事实证明其一贯如此”等无证据动机判断。
- 不写“全网愤怒”“舆论彻底反转”“品牌形象毁于一旦”等无数据绝对化表达。
- 不使用“值得注意的是”“不得不说”“更可怕的是”等AI式强调套话。
- 不虚构现场画面、会议对话、内部决策或人物心理来增强故事性。
- 不用概念堆砌代替对具体事实的解释。

## 6. 成稿质量检查

- [ ] 案例名称、爆发时间、最早公开记录、核心争议和影响范围完整。
- [ ] 企业成立发展、商业模式、市场地位和融资／上市信息有来源且与事件相关。
- [ ] 时间线覆盖首发、企业每次回应、重要媒体、监管、热搜、资本与后续整改。
- [ ] 每个关键时间节点有具体日期、事实描述、影响评估和可点击链接。
- [ ] 权威媒体、社交舆情和专家观点分别梳理，并注明主体、时间和来源。
- [ ] 企业每次重要回应均单独列出并链接原文。
- [ ] 短期回应、沟通策略、立即整改和中长期品牌修复均有分析。
- [ ] 长期修复覆盖价值观、信任、透明度、第三方背书、合规和行业标准。
- [ ] 深层原因覆盖商业模式、管理机制和行业环境，并呈现替代解释。
- [ ] 企业影响与行业影响均有公开数据或明确的数据缺口说明。
- [ ] 经验教训具体到动作、依据、效果、适用条件和边界。
- [ ] 所有事实、数据、外部观点和高风险表述都有出处。
- [ ] 所有原创分析都能回到事实编号或来源编号。
- [ ] 事实、主张、观点、推断和待核实信息没有混写。
- [ ] 重要链接按类别完整汇总，没有虚构链接或只写媒体名称。
- [ ] 主文形成完整论述，不是表格和清单的堆积。
- [ ] 默认只交付一份统一报告，没有“阶段 A／B／C”重复成稿。
- [ ] 案例背景、时间线、回应原文、数据和引语没有在多个章节重复展开。
- [ ] 应对评估、深层原因和影响启示均提供了新分析，不是对事实的换言重述。
- [ ] 信息重合过半的表格已合并或简化，后文已使用交叉引用。
- [ ] 语言专业、丰满、可读且不口语化，没有无证据的戏剧化表达。
- [ ] 成稿可以在人工复核后作为书稿章节素材使用。
