## 1. 输入与停止条件

### 建议输入

尽量取得以下信息；缺项不影响主体识别时直接继续：

- 事件名称
- 已知链接、关键词、人物或账号
- 研究时间范围
- 研究截止时间与时区
- 重点平台、地区或利益相关方
- 是否需要舆情声量或情绪分析

### 必须停止或降级的情形

- 所有公开搜索和网页阅读能力都不可用，且用户要求完整公开资料研究：停止生成，说明无法可靠还原。
- 主体严重歧义：先提不超过 3 个必要问题。
- 只能看到搜索摘要、二手转载或无法核验的截图：将其当作线索，不写成事实。
- 证据不足：写“公开资料不足，无法确认”，不补齐故事。
- 只能使用用户提供的有限材料：声明“本报告仅基于所提供材料，不等同于完整公开资料检索”。

浏览器自动化或某个社交平台原页不可访问，不等于完全无法联网。仍可继续检索公开监管记录、公司网站和专业媒体页面，但必须逐项披露无法直接核验的原始内容。

## 2. 信息状态

对可能引发争议的信息使用以下标签：

| 标签 | 类型 | 判定标准 | 可否进入事实摘要 |
|---|---|---|---|
| `[F]` | 已确认事实 | 有能直接证明该事项的权威原始记录，或由两个及以上相互独立的可信来源交叉验证 | 可以 |
| `[C]` | 当事方主张 | 企业、投诉者、员工、合作方等直接相关方的公开陈述 | 不可以 |
| `[M]` | 媒体报道 | 专业媒体原创采访或调查，但关键事项尚缺原始记录或独立验证 | 不可以 |
| `[O]` | 观点或评价 | 媒体评论、专家判断、公众表达 | 不可以 |
| `[I]` | 分析推断 | 根据已确认事实作出的解释性判断 | 不可以，只能进入分析复盘 |
| `[U]` | 待核实信息 | 网传、匿名爆料、来源冲突、原始内容缺失或证据不足 | 不可以 |

规则：

- 同一信息可随新证据变更状态，但必须在版本记录中说明。
- “官方账号发布”只能直接证明该账号发布过什么；其内容是否属实仍按主张本身的证据判断。
- “媒体报道”不是低质量标签，而是提示读者该信息的确认链条仍依赖媒体采写。
- `[F]` 只覆盖证据能够证明的范围，不向外延伸。

## 3. 来源分级

| 等级 | 来源类型 | 主要用途 | 关键限制 |
|---|---|---|---|
| A | 监管、司法、警方、政府、交易所、企业法定披露等公开记录 | 证明相关机关或法定主体采取了什么行动、作出什么正式认定 | 区分立案、调查、处罚、判决和生效状态，不扩大认定范围 |
| B | 涉事企业、投诉者、合作方、平台等相关方的官方原文 | 证明某主体何时、通过什么渠道表达了什么或宣布了什么行动 | 相关方陈述不等于独立事实认定 |
| C | 有署名、采访过程和原始材料引用的专业媒体原创报道 | 提供采访、现场信息和多方说法 | 转载须回溯首发；匿名信源谨慎使用 |
| D | 有作者、专业身份、数据和方法说明的学术或专业资料 | 提供行业背景、技术解释和基准数据 | 专业身份不代表掌握具体事件内幕 |
| E | 原始帖子、直播、图片、视频、评论或话题页面 | 证明某账号发布过什么及可观察的传播痕迹 | 核验账号、原始上传者、时间、上下文和完整性 |
| F | 聚合页、营销号、自媒体转述、搜索摘要、无出处截图 | 发现线索 | 不得作为关键事实唯一证据 |

### 来源记录规则

- 优先链接原始发布页。
- 每个来源分配唯一编号 `[S01]`、`[S02]`……
- 记录标题、发布主体或作者、发布时间、事件时间、链接、访问日期和状态。
- 页面需登录、已删除、无法访问或仅有摘要时，明确标注限制。
- 发现更正、撤回或更新时，使用最新可核验内容，同时保留变化说明。
- 重要网页易删除时，在合法且工具允许的前提下补充公开存档或截图说明，不得伪造存档。
- 社交平台原帖无法直接打开时，可用两家相互独立、明确引用该原帖的专业媒体确认“媒体曾报道该主体发布过相关内容”。当事方表述仍标为 `[C]`，引用链同时注明媒体来源和“原帖未直接核验”；不得据此声称已经核对完整原文、准确发布时间或后续编辑状态。

## 4. 核验规则

### 4.1 主体消歧

- 核对企业全称、品牌名、母子公司、门店、官方账号和关键人物。
- 说明每个主体的事件角色：涉事方、投诉方、监管方、平台方、合作方、媒体或评论者。
- 企业背景只保留与事件直接相关的信息。

### 4.2 时间核验

- 分开记录事件发生时间和信息发布时间。
- 对图片或视频分开记录疑似拍摄时间与上传时间。
- 标注时区；来源只有日期时，不补写具体时分秒。
- 精确时间不明时使用“约”“此前”“不晚于”等表述并说明依据。
- “首发”“首次回应”“最早出现”必须跨平台回溯；无法穷尽时写“本次公开检索范围内发现的最早记录”。

### 4.3 交叉验证

- 对核心争议、高风险指控、关键节点和重要数据优先查找原始记录，并尽量寻找两个相互独立的可信来源。
- 多家媒体引用同一份声明、同一匿名爆料或同一首发报道，只算一条证据链。
- 单一权威文件可以证明文件明确认定的事项，不能证明文件未处理的延伸主张。
- 只有一个可信来源时标为 `[C]` 或 `[M]`，并写“尚缺独立交叉验证”。
- 来源冲突时建立冲突表，呈现各方说法、支持材料、反向材料和当前确认边界，不强行裁决。

### 4.4 数字内容核验

对关键帖子、截图、图片、音频或视频尽可能核查：

- 发布账号身份及认证状态；
- 最早可找到的版本和原始链接；
- 上传时间与内容发生或拍摄时间；
- 地点、天气、环境、招牌、人物、影像连续性等上下文；
- 是否剪辑、拼接、旧图新用、跨事件挪用或断章取义；
- 原帖是否删除或修改，截图是否缺少账号、时间或上下文。

无法核验时标为 `[U]`。

### 4.5 多方回应

- 收集企业、投诉者或受影响者、监管部门、平台、合作方等关键主体的公开说法。
- 对负面指控检索被指一方回应。
- 未发现回应时写“截至检索截止时间，未在公开渠道检索到回应”。
- 只有来源明确写明联系过程时，才能写“媒体联系后未获回复”等描述。
- 同时呈现支持和削弱重要主张的相关证据，不选择性取材。

### 4.6 舆情数据

只有存在公开数据或能够实际执行合规分析时，才输出声量、情绪占比、趋势或“主流观点”。至少披露：

- 平台和采集工具；
- 关键词、话题标签和排除词；
- 采集时间范围和时区；
- 样本量、采样方式和分析单位；
- 去重、广告、机器人或组织化转发处理；
- 人工或模型情绪分类规则；
- 数据缺口、平台偏差和代表性限制。

没有代表性样本和完整方法时，只写“在所观察样本中出现的意见”，不得写“公众普遍认为”，不得编造百分比。

### 4.7 数据、影响与因果

- 股价、销量、下载量、门店变化、搜索指数等数据注明原始来源、统计口径、比较区间和基准值。
- 时间先后不等于因果。证据不足时只陈述同期变化，说明影响因素可能不唯一。
- “品牌受损”“信任下降”等判断需要调查、业务数据、公开行为或持续舆情证据；否则写成“出现相关质疑”。
- 区分“宣布整改”“开始整改”“完成整改”和“经第三方或监管验证”。

### 4.8 法律、隐私与引用

- 对违法、犯罪、欺诈、造假、侵权、伤亡和财务损失等高风险指控，写明信息来源和认定主体。
- 使用“被指”“某方称”“通报称”“法院判决认定”等与证据状态一致的措辞。
- 不公开与公共利益无关的手机号、住址、身份证号、健康、行踪、生物识别或金融账户信息。
- 涉及未成年人和普通个人时默认匿名化、最小化。
- 只短摘核验所需原话，保持上下文，不大段复制报道或声明。

## 5. 输出模板

### 5.1 研究说明与局限

列出主体识别、研究范围、截止时间、时区、检索平台、主要检索词、无法访问的资料和数据缺口。

### 5.2 已确认事实摘要

用 5—10 条概括核心事实，只允许 `[F]` 信息，每条附来源编号。证据不足时明确写“尚无法确认”。

### 5.3 案例基本信息

|---|---|---|---|
| 案例名称 | 年份＋主体＋事件关键词 |  |  |
| 涉事主体 | 准确名称及关系 |  |  |
| 事件发生时间 | 与发布时间分开 |  |  |
| 最早公开记录 | 限定本次检索范围 |  |  |
| 核心争议 | 描述争议，不预判对错 |  |  |
| 当前进展 | 调查、处罚、判决、整改等 |  |  |

### 5.4 主体关系

| 主体 | 身份或角色 | 与事件关系 | 依据 | 来源 |
|---|---|---|---|---|

### 5.5 详细时间线

| 事件发生时间 | 信息发布时间 | 可核实节点 | 信息状态 | 涉及主体 | 来源 |
|---|---|---|---|---|---|

覆盖公开资料中实际存在的首发线索、原创报道、当事方回应、平台措施、监管或司法介入、整改、处罚、判决、业务变化和后续更正。每行至少一个来源编号。

### 5.6 各方公开说法与行动

| 主体 | 时间 | 原话短摘或准确转述 | 宣布或采取的行动 | 信息状态 | 来源 |
|---|---|---|---|---|---|

只还原说法和行动，不在此处评价是否及时、真诚或有效。

### 5.7 信息传播与舆情表现

说明传播起点、扩散节点、主要议题、观点分布、合格的声量或情绪数据及代表性限制。不要机械凑媒体数量。

### 5.8 已核实影响

| 影响类别 | 已核实变化 | 时间或比较区间 | 能否确认因果 | 信息状态 | 来源 |
|---|---|---|---|---|---|

按实际证据选择监管法律、业务、合作渠道、资本财务、用户员工、品牌舆情等类别。

### 5.9 争议与待核实事项

| 待核实主张 | 提出者及来源 | 支持证据 | 冲突或反向证据 | 当前确认边界 | 仍需证据 |
|---|---|---|---|---|---|

### 5.10 事实主张—证据台账

| 主张编号 | 具体表述 | 信息状态 | 主要证据 | 独立交叉来源 | 可否进入事实摘要 | 备注 |
|---|---|---|---|---|---|---|
| C01 |  | `[F/C/M/O/I/U]` | `[S01]` | `[S02]` 或“无” | 是／否 |  |

### 5.11 来源索引

| 来源编号 | 等级 | 标题或文件名 | 发布主体或作者 | 发布时间 | 事件时间 | 原始链接 | 访问日期 | 状态与备注 |
|---|---|---|---|---|---|---|---|---|

### 5.12 版本与更正记录

| 版本 | 更新时间 | 新增或更正内容 | 原因 | 影响的结论 | 新来源 |
|---|---|---|---|---|---|

首次报告也建立 v1.0 记录。新增权威信息或发现错误时公开记录变化。

### 5.13 阶段性简报

用户只要求部分章节时，可以先交付指定内容，但末尾必须追加：

1. 精简事实主张—证据台账；
2. 本次实际使用的来源索引；
3. 未通过或尚未执行的事实门槛；
4. 原始社交页面、付费墙、登录限制或其他无法直接核验事项；
5. 下一步需要补充的证据。

阶段性简报不得使用“完整还原”“最终结论”等表述。

## 6. 质量清单

- [ ] 标明研究范围、截止时间和时区。
- [ ] 完成主体消歧。
- [ ] 区分事件时间和发布时间。
- [ ] 每个关键事实和高风险表述均有来源。
- [ ] 优先使用原始发布页并回溯转载上游。
- [ ] 清楚区分事实、主张、报道、观点、推断和待核实信息。
- [ ] 识别多家转载同一来源的伪交叉验证。
- [ ] 呈现重要反向证据、冲突和被指方回应。
- [ ] 披露社交媒体样本与情绪分析方法限制。
- [ ] 避免把时间相关性写成因果。
- [ ] 最小化处理个人信息和未成年人信息。
- [ ] 记录删除、更新、更正、无法访问和数据缺口。
- [ ] 删除无法追溯、夸张、戏剧化或越过证据边界的表述。
